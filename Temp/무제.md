  왜 안 되는가                                                                                                                                                         
                                                                                                                                                                       
  GeofenceOrchestrator는 Riverpod @Riverpod(keepAlive: true) 노티파이어로, 메인 Dart 아이솔레이트에만 존재합니다. 현재 구성은 다음이 누락:                             
                                                                                                                                                                       
  1. foreground service / background isolate 부재                                                                                                                      
    - pubspec.yaml에 flutter_foreground_task, flutter_background_service, workmanager, native_geofence 등 어떤 백그라운드 패키지도 없음.                               
    - geolocator만 단독 사용.                                                                                                                                          
  2. LocationMonitoringService가 plain LocationSettings 사용 (location_monitoring_service.dart:25-28)                                                                  
    - Android 전용 AndroidSettings.foregroundNotificationConfig 미설정 → Android 8+에서 OS가 곧 강제 종료.                                                             
    - iOS 전용 AppleSettings.allowBackgroundLocationUpdates / pauseLocationUpdatesAutomatically 미설정.                                                                
  3. Android 매니페스트에 <service> 선언 없음
    - ACCESS_BACKGROUND_LOCATION, FOREGROUND_SERVICE_LOCATION 퍼미션은 선언되어 있으나 정작 서비스 태그가 없음.
    - BOOT_COMPLETED 리시버도 없어 재부팅 후 복원 불가.
  4. iOS는 UIBackgroundModes=location이 있어 포그라운드 → 백그라운드 전환 직후 일정 시간만 살아남음. 앱 스와이프/OS suspend 시 종료.
  5. process kill 복원 로직 없음: 앱이 죽으면 ProviderContainer가 사라지고 startMonitoring을 호출할 주체가 없음.

  결론: 현재는 "앱이 포그라운드거나 OS 메모리 상에 살아있는 동안만" 동작. 사용자 스와이프·Doze·재부팅·OEM 배터리 정책 어느 하나라도 개입하면 실패.

  ---
  계획 (구현은 하지 않음)

  세 가지 노선 중 하나를 선택해야 함.

  A안 — OS 네이티브 geofence API로 이전 ⭐ 권장

  OS가 지오펜스 진입 이벤트를 직접 감지하고 앱을 깨워줌. 앱 종료 상태에서도 동작, 배터리 최적.

  작업 범위
  6. native_geofence 또는 동급 패키지 추가.
  7. 백그라운드 엔트리포인트 (@pragma('vm:entry-point') 최상위 함수) 작성 — 아이솔레이트 전용 DI 초기화, 수신자 조회, SMS/FCM 발송, 레코드 저장, 비활성화까지 수행.
  8. NativeGeofenceRegistrar 서비스 신설 — save / updateActiveStatus / delete 훅에서 OS에 register/unregister.
  9. Android: RECEIVE_BOOT_COMPLETED 리시버 추가 (재부팅 후 재등록).
  10. iOS: 리전 20개 제한 → 활성 지오펜스 20개 초과 시 근접도/우선순위 기반 사이클 정책 필요.
  11. 기존 GeofenceOrchestrator/LocationMonitoringService/GeofenceCheckingService/DeduplicationService는 대부분 폐기 또는 "현장 테스트 모드" 전용으로 축소. OS가
  체크/dedup을 담당하므로 중복.

  트레이드오프: 아키텍처 파급 큼. 배터리·신뢰성 최고.

  B안 — Foreground service로 현재 구조 래핑

  연속 스트림 구조 유지, OS 종료 방지를 위해 persistent notification 서비스로 감쌈.

  작업 범위
  12. flutter_foreground_task 추가.
  13. _checkGeofences 루프를 task의 onRepeatEvent로 이관.
  14. geolocator를 AndroidSettings(foregroundNotificationConfig: ...) + AppleSettings(allowBackgroundLocationUpdates: true)로 교체.
  15. AndroidManifest에 <service android:foregroundServiceType="location" ...> 추가.
  16. BOOT_COMPLETED 리시버로 재부팅 복원.
  17. 사용자에게 상시 알림이 보이는 UX 부담 존재.

  트레이드오프: 코드 보존률 높음. 배터리 소모·UX 비용 큼.

  C안 — 하이브리드 (큰 반경 OS geofence + 근접 시 foreground 전환)

  근접 영역에서만 foreground 스트림 활성. 정확도·배터리 균형 최상, 복잡도도 최상.
